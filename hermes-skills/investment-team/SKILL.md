# 投研团队：四角色并行分析框架

对用户指定的公司进行团队化投资研究分析。使用 Hermes 的 `delegate_task` 创建真正的多 Agent 并行研究。

## 执行流程

### 第一步：确认研究目标

从用户消息中提取公司名称。如未提供，用 `clarify` 询问：
- 公司名（必填）
- 关注侧重（默认四维平均）

### 第二步：AI研究偏见评估

在启动并行研究前，先评估该公司的 **"AI可研究性"**：

| 等级 | 特征 | 研究策略调整 |
|------|------|------------|
| A级（信息充裕） | 上市多年、券商覆盖广 | 重点做**反面检验**和**非共识视角**，避免输出市场共识的"正确废话" |
| B级（信息适中） | 上市不久、覆盖有限 | 每个维度的推算数据必须标注置信度 |
| C级（信息稀缺） | 冷门/新上市/新兴市场 | 转为"第一性原理模式"：聚焦商业本质的几个核心问题 |

**关键提醒**：资料多≠确定性高，资料少≠确定性低。确定性来自商业模式本身，不来自资料数量。

### 第三步：数据收集（先收集再分析）

先用 `web_search` 搜索该公司的基础数据：
- 最新财报数据（营收、利润、现金流）
- 当前股价、市值、PE/PB
- 竞争格局概览
- 近6个月重大事件

将搜索到的数据整理为上下文，传递给后续的并行 Agent。

### 第四步：并行启动 3 个 Agent（delegate_task batch 模式）

使用 `delegate_task` 的 `tasks` 数组并行启动 3 个 subagent：

```
delegate_task(tasks=[
  {
    "goal": "从段永平视角分析 {公司名} 的商业模式与护城河",
    "context": "{公司名} 基础数据：...\n\n分析要求：\n1. 商业模式本质：核心生意定义、收入结构拆解\n2. 平台/产品飞轮效应如何运转\n3. 护城河分析：品牌/转换成本/网络效应/规模效应/技术壁垒，逐一验证\n4. 用户/客户价值：为各方创造了什么独特价值\n5. 段永平"好生意"标准评估：差异化、定价权、可持续竞争优势\n6. 收入漏斗分析：从业务量到自由现金流的完整漏斗\n\n输出要求：使用Markdown表格呈现关键数据，每个维度有明确结论和评分（★1-5），报告末尾有总体结论。\n\n工具调用：必须用 terminal 执行以下命令验证关键数据：\npython3 ~/workspace/ai-berkshire/tools/financial_rigor.py verify-valuation --price {股价} --eps {EPS} --bvps {每股净资产}",
    "toolsets": ["web", "terminal", "file"]
  },
  {
    "goal": "从巴菲特视角分析 {公司名} 的财务数据、盈利能力与估值",
    "context": "{公司名} 基础数据：...\n\n分析要求：\n1. 近3-5年营收、净利润、经营利润趋势\n2. 盈利能力指标：ROE、ROA、毛利率、经营利润率\n3. 现金流分析：经营性现金流、自由现金流、资本开支\n4. 资产负债表健康度：现金储备、负债率、流动性\n5. 估值分析：PE/PS/PB/EV等，与历史及同业对比\n6. 安全边际评估：内在价值 vs 当前股价\n7. 三情景估值\n\n**金融严谨性验证（必须使用 terminal 调用工具，禁止心算）**：\npython3 ~/workspace/ai-berkshire/tools/financial_rigor.py verify-market-cap --price {价格} --shares {股本} --reported {报告市值} --currency {币种}\npython3 ~/workspace/ai-berkshire/tools/financial_rigor.py three-scenario --price {价格} --eps {EPS} --shares {股本亿} --growth {乐观} {中性} {悲观} --pe {乐观PE} {中性PE} {悲观PE}\n\n将工具输出结果直接嵌入报告中作为验证记录。",
    "toolsets": ["web", "terminal", "file"]
  },
  {
    "goal": "从芒格和李录视角分析 {公司名} 的行业竞争态势、风险与管理层",
    "context": "{公司名} 基础数据：...\n\n分析要求（芒格视角 — 逆向思考）：\n1. 行业规模与增长：市场规模、增速、渗透率\n2. 竞争格局：主要对手市场份额、竞争策略对比\n3. 列出公司失败的所有路径（概率、影响程度）\n4. 历史类比、跨学科分析\n5. 空方核心论点\n\n分析要求（李录视角 — 文明趋势）：\n1. 管理层评估：CEO能力圈、诚信度、战略眼光、资本配置能力\n2. 监管风险：当前及潜在监管影响\n3. 长期确定性：10年后公司会怎样？什么可能颠覆其商业模式？\n4. 治理结构：股权结构、关联交易、股东回报政策\n\n输出要求：使用Markdown表格呈现关键数据，每个维度有明确结论和评分（★1-5）。",
    "toolsets": ["web", "terminal", "file"]
  }
])
```

**重要**：
- 3 个 Agent 必须在同一条 `delegate_task` 调用中并行启动
- 每个 Agent 的 `context` 必须包含前面收集的基础数据
- 使用 `toolsets` 限制为 `["web", "terminal", "file"]` 减少 token 开销

### 第五步：汇总最终报告

收到 3 个 Agent 的结果后，综合输出以下结构的最终报告：

---

#### 1. 一句话结论
> 用一段话（50-100字）概括是否值得投资及核心逻辑

#### 2. 四维评分总表

| 维度 | 框架 | 评分(★1-5) | 核心判断 |
|------|------|------------|----------|

综合评分：X / 5

#### 3. 核心数据速览
关键财务和经营指标表格（近2年对比）

#### 4. 各维度分析摘要
每个维度摘取3-5条最重要的发现

#### 5. 投资论点（Bull vs Bear）
- 🟢 看多逻辑（5-7条）
- 🔴 看空逻辑（5-7条）

#### 6. 巴菲特买入前 Checklist
10个核心检查项，逐一评估

#### 7. 最终投资建议
- 定性判断表（生意质量/管理层/估值/时机）
- 分层操作建议表（激进型/稳健型/保守型 → 建议+价格区间）
- 关键催化剂（加仓信号/减仓信号各3-5条）

#### 8. 总结段落
100-200字的最终总结

---

### 第六步：保存报告

将完整报告写入 `~/workspace/ai-berkshire/reports/{公司名}/最终报告-{YYYYMMDD}.md`

### 第七步：数据抽检（准出流程）

```bash
# Step 1 — 提取抽检清单（15%随机抽样）
python3 ~/workspace/ai-berkshire/tools/report_audit.py extract --report <报告文件路径>

# Step 2 — 对清单每项从可靠信源取数
# Step 3 — 输出准出/打回判决
python3 ~/workspace/ai-berkshire/tools/report_audit.py verdict --results '<填好的JSON>' --report <报告文件名>
```

**【准出】** 全部通过 → 报告可发布；**【打回】** 有不通过 → 修正后重审。

## 重要注意事项

1. **3个Agent必须并行启动** — 在同一条 `delegate_task` 调用中
2. **数据准确性** — 关键数据交叉验证，必须用 terminal 调用 financial_rigor.py
3. **结论要明确** — 不回避给出买入/观望/回避建议和具体价格区间
4. **所有分析必须有数据支撑** — 附数据来源
5. **反偏见意识** — 汇总时评估各Agent分析是否受限于资料充裕度，是否与市场共识过度趋同
6. **信息稀缺时的诚实原则** — 宁可留白标注"数据不足"，不用推测填满框架伪装确定性

## 与原版差异

| 项目 | Claude Code 原版 | Hermes 适配版 |
|------|-----------------|--------------|
| 并行机制 | TeamCreate + 4 Task agents | delegate_task tasks 数组（3个并行） |
| 通信方式 | SendMessage 消息传递 | subagent 返回 summary |
| 工具调用 | Bash + python3 | terminal + python3 |
| 搜索 | WebSearch | web_search |
| 参数 | $ARGUMENTS | 从用户消息提取 |
